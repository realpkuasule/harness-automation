## 3. Remaining Observations

### 3.1 `CognitiveResponse` — Discriminator Representation (Documentation Format)

| Source | Mechanism |
|--------|-----------|
| **Spec** | OpenAPI `oneOf` + `discriminator.propertyName: skillType` |
| **Impl** | TypeScript discriminated union `type CognitiveResponse = DiagnosticResult \| EducationalResult \| DecisionSupportResult` |

This is a documentation format difference, not a behavioral gap. The actual response payloads match exactly:

- **DiagnosticResult**: `{ skillType: "diagnostic", analysis, severity, rootCause, fixSuggestion, relatedRules }` ✅
- **EducationalResult**: `{ skillType: "educational", explanation, bestPractices, examples, learningResources }` ✅
- **DecisionSupportResult**: `{ skillType: "decision-support", tradeoffs, recommendation, alternatives }` ✅

### 3.2 `evaluate_rules` — Two-State Saves (Implementation Detail)

The spec describes saving to `.harness/state.json` in one step. The implementation calls `stateManager.setEngineInput(input)` then `stateManager.setEngineOutput(output)` separately, each triggering a full `writeFileSync`. On dry-run, no state is written — matching spec.

**Impact**: None. Observable behavior matches spec.

### 3.3 `start_ab_test` — `endTime` for `already_running` Status

When `status === "already_running"` and the existing test has expired, `endTime` is set to `existing.createdAt` (manager.ts:64-65). For non-expired running tests, `endTime` is `undefined`. This is consistent with the spec note "仅 status=started 时存在" since `already_running` is not `started`.

### 3.4 Internal Type Notes

These are not spec-impl gaps:

- **`RuleExportData` omits enriched fields**: On export, computed fields (`errorMessage`, `adjustedCost`, `adjustedCostLabel`, `feedbackSpeed`) are not preserved. On re-import they're reconstructed from rule definitions. Lossy but does not affect the API contract.

- **`HarnessState.validation.findings` computed**: `state.validation.findings` is computed as `errors + warnings + info`. The spec shows `findings: integer`, consistent. ✅
