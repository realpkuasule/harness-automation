## Gap 1: `HarnessState.evaluatedAt` Missing from Spec

**Severity**: Low (documentation omission)
**Affects**: `components/schemas/HarnessState`
**Origin**: P13-9 fix (added `evaluatedAt` to implementation but not to spec)

### Description

P13-9 added `evaluatedAt?: string` to `types.ts:229` (`HarnessState` interface) and stores it in `state.ts` at `setEngineInput()` and `setEngineOutput()`. However, the OpenAPI spec's `HarnessState` schema does not include this field.

### Evidence

**Spec `HarnessState`** (lines ~2235-2260):
- `phase`, `projectDir`, `createdAt`, `updatedAt`, `engineInput`, `engineOutput`, `decisions`, `confirmedAt`, `configOutput`, `version`, `sessionId`, `validatedAt`, `generationLog`, `validation`, `project`
- **No `evaluatedAt` field**

**Implementation `HarnessState`** (`types.ts:221-249`):
- All of the above + **`evaluatedAt?: string`** (line 233, added in P13-9)

**`QueryStateOutput.lastEvalAt`** (spec lines 1606-1608):
- Uses `lastEvalAt` (derived from `state.evaluatedAt ?? state.updatedAt` in `index.ts:377`)
- This output field exists correctly, but the **source** field `evaluatedAt` is missing from the `HarnessState` schema

### Fix

Add `evaluatedAt` to the spec's `HarnessState` schema:

```yaml
evaluatedAt:
  type: string
  format: date-time
  description: 最近一次评估时间（evaluate_rules 或 scan_codebase 设置）
```

Insert after `updatedAt` (~line 2246).

## Gap 2: `optimize_error_message.projectDir` Required vs Optional Mismatch

**Severity**: Medium (behavioral — Zod rejects valid calls)
**Affects**: `/tools/optimize_error_message` input schema
**Origin**: Pre-existing (missed in prior analysis)

### Description

The **spec** says no parameters are required for `optimize_error_message` (no `required` array in the request body). The P3-5 edge case description added in P13-12 explicitly states "所有参数可选。不提供 ruleId 或 scenario 时 suggestions 为空数组。"

The **handler** treats `projectDir` conditionally:
- Line 1183: `if (optInput.projectDir && optInput.rateAfter !== false)` — evaluation recording
- Line 1193: `if (optInput.projectDir && optInput.ruleId)` — trigger history

This confirms the handler logic supports `projectDir` being absent.

However, the **Zod schema** (`OptimizeErrorMessageInputSchema`, `types.ts:505`) defines `projectDir: z.string()` — **required**, not optional. Calling the tool without `projectDir` will be rejected by Zod before the handler runs.

### Evidence

| Source | `projectDir` | `ruleId` | `scenario` |
|--------|-------------|----------|------------|
| Spec request body | optional | optional | optional |
| Zod schema (`types.ts:505`) | **required** | optional | optional |
| Handler logic | conditional checks | conditional checks | used directly |

### Fix

Make `projectDir` optional in the Zod schema to match the spec description and handler logic:

```typescript
// types.ts — OptimizeErrorMessageInputSchema
projectDir: z.string().optional(),
```
