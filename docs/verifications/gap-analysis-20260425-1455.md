## 4. 响应格式差异

### 4.1 MCP 消息包装

已通过 P8-13 (`x-mcp-note`) 在 OpenAPI 中说明。无功能差距。

### 4.2 错误响应格式

所有 12 个错误返回点均使用 `{error: {code, message, recoverable}}` + `isError: true` 格式。

**注意**: `cognitive_skill` handler (`index.ts:1224`) 的错误响应缺少 `null, 2` 格式化参数，其他错误均带 pretty-printing:
```typescript
// cognitive_skill (line 1224) — 无 pretty-print
JSON.stringify({ error: { code: "UNKNOWN_ERROR", message: String(e), recoverable: true } })

// 其他工具 — 有 pretty-print
JSON.stringify({ error: { code: "...", message: "...", recoverable: true } }, null, 2)
```

级别: 低（仅格式化差异）。

## 5. 行为 / 实现细节差异

### 5.1 generate_config action 枚举值 `create` vs `created`

OpenAPI (`ConfigFileResult.action` line 1529) 枚举为 `[create, overwritten, skipped, merged, dry_run]`，与实现一致。但 OAS 注释（line 1532）指出设计使用过去分词（`created`），这是追踪中的已知偏差。

### 5.2 `adjustedCostLabel` 字段

engine.ts `_adjustCost()` 返回 `{value, label}`，但 OAS `RuleDecision` 中未定义 `adjustedCostLabel` 字段。差距已在 P6-16 中追踪（pending）。
