## 一、P0 — 关键回归（Phase 8 修复引入的新错误）

### GAP-01: `installNote` 在 OpenAPI 中放置位置错误 — 应在 `InitHarnessSummary` 顶层而非 `permissions` 下

| 属性 | OpenAPI | 实现 |
|------|---------|------|
| installNote 位置 | `InitHarnessSummary.permissions.installNote` (L2399) | `summary.installNote` — 顶层字段 (index.ts L706) |

**分析**: Phase 8 P8-1 修复本应将 `installNote` 加入 `InitHarnessSummary`，但错误地放在了 `permissions` 子对象内部。实现代码中 `installNote` 与 `permissions` 是平级的顶层字段。

**实现代码** (`index.ts:697-707`):
```typescript
const summary = {
  files: fileSummary,
  decisions: evalOutput.summary.total,
  backupDir: backupDir || null,
  recommendedActions: depInfo,
  huskySetup: ...,
  permissions: permissionHint,    // { note, command, suggestedTools }
  installNote: "Installing dependencies may take 2-5 minutes...",  // ← 顶层!
};
```

**OpenAPI 当前** (L2384-2401):
```yaml
permissions:
  type: object
  properties:
    note: ...
    command: ...
    suggestedTools: ...
    installNote:           # ← 错误：放在了 permissions 下
      type: string
      description: 依赖安装耗时提示
```

**应修改为**: `installNote` 移到 `InitHarnessSummary` 的顶层 properties，与 `permissions` 平级。

**涉及文件**: `docs/api/openapi.yaml` L2384-2401

## 二、P1 — 输出 Schema 缺失（变体输出未文档化）

### GAP-02: `rollback` list 模式输出未在 OpenAPI 中定义

| 属性 | OpenAPI | 实现 |
|------|---------|------|
| list=true 响应 | 引用 `RollbackOutput` (L350) | 返回 `{ backups: Array<{id, files, createdAt}> }` (index.ts L817-824) |

**分析**: 当 `rollback` 以 `list=true` 调用时，实现返回完全不同的输出形状（backup 列表），而非 `RollbackOutput`（restore 结果）。OpenAPI 仅定义了 restore 模式的响应 schema。

**实现代码** (`index.ts:816-825`):
```typescript
if (input.list) {
  return {
    content: [{
      type: "text",
      text: JSON.stringify({ backups: backupList }, null, 2),
    }],
  };
}
```

**涉及文件**: `docs/api/openapi.yaml` L328-350

### GAP-03: `init_harness` dryRun 模式输出变体未在 Schema 中体现

| 属性 | OpenAPI | 实现 |
|------|---------|------|
| dryRun=true 响应 | Schema 始终包含 `files`, `summary: InitHarnessSummary`, `validation`, `errors` (L259-273) | 返回 `{ files: [], summary: { files: {...}, decisions: N } }` — 无 `validation`、`errors`、且 `summary` 不含 `backupDir`/`recommendedActions`/`huskySetup`/`permissions`/`installNote` (index.ts L492-506) |

**分析**: OpenAPI L235-236 description 中正确描述了 dryRun 行为差异，但 Response Schema (L259-273) 将所有字段都标为存在。`validation` 和 `errors` 在 dryRun 时不存在，`InitHarnessSummary` 中的 `backupDir`/`recommendedActions`/`huskySetup`/`permissions`/`installNote` 在 dryRun 时也不存在。

**涉及文件**: `docs/api/openapi.yaml` L235-236（description 正确），L259-273（Schema 需标注条件性）

## 五、关于 @Roles / 权限注解

本项目为 MCP (Model Context Protocol) 工具规范，通过 stdio 传输调用（非 HTTP REST API）。MCP 协议层不定义角色/权限注解机制。OpenAPI 规范中的 `x-mcp-note` 及 `servers[0].url: "mcp://localhost"` 已标注此为 MCP 接口。

**结论**: 本项目不适用 @Roles 注解。无需修复。
