          type: object
          properties:

              type: integer
            totalLines:
              type: integer

              type: integer
            suggestions:
              type: array
              items:
                $ref: "#/components/schemas/ScanSuggestion"
        extractedRules:
          type: integer
        decisions:
          type: array
          items:
            $ref: "#/components/schemas/RuleDecision"

## P0 单项评估：方案 A vs 方案 B 的比较

由于 P0-2（scan_codebase 输出结构不匹配）存在两种修复路径，在此做对比评估：

| 维度 | 方案 A：改 OpenAPI 匹配代码 | 方案 B：改代码匹配 OpenAPI |
|------|---------------------------|--------------------------|
| 改动范围 | 1 个文件（openapi.yaml） | 3 个文件（integration.ts, index.ts, code_scanner.ts） |
| 影响已有客户端 | 无 | 已读取 scan_codebase 的客户端需适配新格式 |
| 实现复杂度 | 低（仅文档调整） | 高（重构返回结构，拆分 findings/existingRules/suggestedRules） |
| 与设计文档对齐度 | 低（偏离设计的 3 段式） | 高（与设计 §3.1.6 一致） |
| 未来扩展性 | 如需实现 3 段式，OpenAPI 需再改一次 | 一次到位 |

**推荐: 方案 A**（改 OpenAPI）。当前实现的扁平结构实际可用，设计的三段式是未来优化方向，不应在修复阶段强行对齐设计。
