## P1 Items

### P1-1: `optimize_error_message.projectDir` Required by Zod But Spec/Handler Treat as Optional

**History**: New finding (first appeared in 2026 report, §Gap 2)
**Source**: `gap-analysis-20260425-2026.md §Gap 2`
**Recurring**: No

**Root cause**: `types.ts:506` defines `projectDir: z.string()` (required) in `OptimizeErrorMessageInputSchema`, but the handler (`index.ts:1183,1193`) uses conditional checks (`if (optInput.projectDir && ...)`), and the spec says `required: []` with description "所有参数可选". Callers who omit `projectDir` get a Zod validation error before the handler runs, even though the handler would work fine without it.

**Impact**: Low in practice (MCP clients usually send projectDir), but the type system is stricter than the runtime behavior, which violates the "spec is source of truth" contract.

**Files to modify**:

| # | File | Line | Change |
|---|------|------|--------|
| 1a | `mcp-server/src/types.ts` | 506 | Change `z.string()` → `z.string().optional()` for `projectDir` in `OptimizeErrorMessageInputSchema` |

**Verification**:
1. Run `npx vitest run` — all 320+ tests must pass
2. Confirm Zod no longer rejects calls without `projectDir`
3. Confirm handler's conditional branches (`optInput.projectDir && ...`) still work correctly when `projectDir` is absent

## P2 Items

### P2-1: `HarnessState.evaluatedAt` Missing from OpenAPI Spec

**History**: New finding (first appeared in 2026 report, §Gap 1; originated from P13-9 code fix)
**Source**: `gap-analysis-20260425-2026.md §Gap 1`
**Recurring**: No

**Root cause**: P13-9 added `evaluatedAt?: string` to `types.ts:232` (`HarnessState` interface) and stores it in `state.ts:74,84` at `setEngineInput()`/`setEngineOutput()`. The spec's `HarnessState` schema was not updated. The output field `QueryStateOutput.lastEvalAt` is correctly described (it derives from `state.evaluatedAt ?? state.updatedAt` at `index.ts:377`), but the intermediate storage field `evaluatedAt` in `HarnessState` is undocumented.

**Files to modify**:

| # | File | Line(s) | Change |
|---|------|---------|--------|
| 2a | `docs/api/openapi.yaml` | ~2248 (after `updatedAt` field) | Add `evaluatedAt` field to `HarnessState` schema |

Add this block:

```yaml
        evaluatedAt:
          type: string
          format: date-time
          description: 最近一次评估时间（evaluate_rules 或 scan_codebase 设置）
```

**Verification**:
1. Read the modified spec — `HarnessState` now includes `evaluatedAt`
2. Confirm `QueryStateOutput.lastEvalAt` (line 1606) is still consistent: it derives from `evaluatedAt`, not `updatedAt`
3. No code change needed
