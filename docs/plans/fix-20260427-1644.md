## 1. ESLint 配置格式：从 `.json` 改为 `.js`（Flat Config）

### 问题

- 当前生成 `eslint.config.json`，但 ESLint v9+ 已废弃 `.eslintrc*` 格式，默认只加载 `eslint.config.js`（或 `.mjs`、`.ts`）
- 生成的内容是 `{ "rules": { ... } }` 的旧版格式，而非 flat config 的 `export default [ ... ]` 格式
- CI 命令 `npx eslint . --max-warnings=0` 不会自动加载 `.json` 文件

### 修改文件

- `mcp-server/src/generators/eslint.ts` — 改为输出 `export default [{ rules: { ... } }];` 的 JS 模块格式
- `mcp-server/src/generators/eslint.test.ts` — 更新测试断言
- `mcp-server/src/index.ts` L1305 — 文件名从 `eslint.config.json` 改为 `eslint.config.js`
- `mcp-server/src/validators/setup_validator.ts` L44 — `MANAGED_FILES` 列表中路径更新
- `mcp-server/src/validators/setup_validator.ts` L60 — `REQUIRED_DEV_DEPS` 中 key 更新
- `docs/api/openapi.yaml` — `ValidateSetupInput` 的 default checkFiles 更新

### 注意事项

- flat config 格式需要 `export default` 语法（ESM），但项目是 CommonJS——所以输出 `.js` 时可以用 `module.exports = [...]`，或使用 `.mjs`
- 建议使用 `eslint.config.js` + `module.exports = [...]`（兼容 CommonJS）

## 2. ESLint 规则前缀：从 `typescript-eslint/` 改为 `@typescript-eslint/`

### 修改文件

- `mcp-server/src/generators/eslint.ts` L38、L41 — 规则名前缀更新
- `mcp-server/src/generators/eslint.test.ts` L75-76 — 测试期望值更新

## 3. `.husky/pre-commit` 内容过简

### 问题

- 当前生成的 pre-commit 只包含 `npm test`（9 字节）
- R008（lint-before-commit → hook）要求 commit 前执行 lint
- 应有 `npx lint-staged` 或 `npx eslint . --max-warnings=0`

### 修改文件

- `mcp-server/src/generators/husky.ts` L37-39 — pre-commit 应包含 `npx lint-staged`（而不是直接在 hook 中跑 eslint），由 `lint-staged` 配置决定具体执行内容

### 补充依赖

- `package_json.ts` — RULE_DEPS 中 `"lint-before-commit"` 已包含 `lint-staged`，正确
- 需在 `generateProjectFiles` 或 `init_harness` 中补充生成 `.lintstagedrc.json`（或放入 package.json）

## 4. `.husky/commit-msg` 未生成

### 问题

`init_harness` 调用 `generateHuskyConfig` 时会生成 commit-msg hook（代码逻辑正确），但实战中最终验证时该文件缺失。需要排查 `generateProjectFiles` 与 `init_harness` 的文件生成逻辑：

- `generateProjectFiles()`（L1292）只生成 4 个核心文件，不包含 husky
- `init_harness`（L523）单独调用 `generateHuskyConfig` 并写入

### 排查方向

- 确认 `init_harness` 中 husky 文件写入后是否被后续操作覆盖
- 确认 `init_harness` 的写入路径是否正确（L573 的循环是否覆盖到了 `.husky/` 文件）

## 6. CI 工作流 ESLint 调用路径

### 问题

- `npx eslint . --max-warnings=0` 需要配置文件参数才能加载 `eslint.config.js`
- 如果使用 ESLint v9 flat config，应指定 `--config eslint.config.js` 或确保 eslint 能自动发现

### 修改文件

- `mcp-server/src/generators/ci.ts` L57 — 根据 ESLint 配置文件名调整命令：
  ```yaml
  # 当前（可能不加载自定义配置）:
  run: npx eslint . --max-warnings=0
  # 改为:
  run: npx eslint --config eslint.config.js . --max-warnings=0
  ```
