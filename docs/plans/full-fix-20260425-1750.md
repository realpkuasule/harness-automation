| 属性 | 内容 |
|------|------|
| **文件** | `docs/api/openapi.yaml` |
| **行号** | L681-685 |
| **问题** | Spec 将 `metrics` 定义为 `type: string` 数组，但实现（`mcp-server/src/types.ts:479`）接受联合类型：`string[] | {name: string, weight: number}[]`。实现向后兼容 string 数组，但 spec 不反映完整的类型范围 |

```yaml
                  type: array
                  items:
                    type: string
                  default: [triggerCount, fixRate, bypassCount]
```

改为：

```yaml
                metrics:
                  type: array
                  items:
                    oneOf:
                      - type: string
```

| **验证方法** | 确认 spec 的 YAML 结构合法；确认生成的 JSON Schema 包含 oneOf | P1 |

```yaml
              schema:
                oneOf:
                  - title: PresetImportResult
                    type: object

                      preset:
                        type: string
                      presetName:
                        type: string
                      decisions:
                        type: array

                          $ref: "#/components/schemas/RuleDecision"

                  - title: RawImportResult

            type: string
```

```yaml
          type: array

            type: object
```

## P2 — 行为改进（建议修复）

### P2-1: `init_harness` 响应不暴露文件写入错误

| 属性 | 内容 |
|------|------|
| **文件** | `mcp-server/src/index.ts` L567-579（实现）；`docs/api/openapi.yaml` L238-252（spec） |
| **问题** | `init_harness` handler 在写文件循环中捕获异常并存入 `errors` 数组（L569-578），但该数组从未被包含在 response 中。错误仅通过 `sm.setConfigOutput({ ..., errors })` 保留在 state 中。调用者无法在响应中看到哪些文件写失败了 |
| **来源** | 报告 §4.3 — **新发现** |
| **修复方案** | 两方面修复：在 response 中加入 `errors` 字段；在 spec 中补充该字段声明 |
| **具体修改** | **(A) Spec 修改** — 在 L250-251 之间加入： |

```yaml
                  errors:
                    type: array
                    items:
                      $ref: "#/components/schemas/ConfigError"
                    description: 文件写入操作中发生的错误列表（仅非 dryRun 模式）
```

**(B) 实现修改** — 在 `mcp-server/src/index.ts` L589（`const output = { files, summary }`）处将 `errors` 纳入 response：

将：
```typescript
const summary = { ... };
const output = { files, summary };
```
改为：
```typescript
const summary = { ... };
const output = { files, summary, errors: errors.length > 0 ? errors : undefined };
```

| **验证方法** | 单元测试：模拟 `writeFileSync` 抛出异常，检查 `init_harness` 响应包含 `errors` 数组 | P2 |
