## P1-1: UNKNOWN_TOOL 未按规范返回结构化错误

### 来源
gap-analysis-20260427-2111.md M1 + 本次验证确认

### 历史
- **首次发现**，未在既往报告中出现过

### 当前状态验证
```bash
$ grep -rn "UNKNOWN_TOOL" mcp-server/src/
mcp-server/src/types.ts:61:  | "UNKNOWN_TOOL";     # 定义在 ErrorCode 枚举中
$ grep -rn "UNKNOWN_TOOL\|unknown tool\|Unknown tool" mcp-server/src/index.ts
1220:      throw new Error(`Unknown tool: ${name}`);  # 抛出通用 Error，非结构化 HarnessError
```

OpenAPI 中 `ErrorCode` 枚举包含 `UNKNOWN_TOOL`，并且 `HarnessError` 是所有 400 响应的标准格式。但 `index.ts` default case 使用 `throw new Error()`，MCP SDK 将其转为协议级错误，而非结构化 `HarnessError`。

### 改动

| 文件 | 行号 | 当前内容 | 改为 | 说明 |
|------|------|----------|------|------|
| `mcp-server/src/index.ts` | L1219-1221 | `default:\n  throw new Error(\`Unknown tool: ${name}\`);` | `default:\n  return { content: [{ type: "text", text: JSON.stringify({ code: "UNKNOWN_TOOL", message: \`Unknown tool: ${name}\`, detail: \`The tool "${name}" is not registered\`, recoverable: false })], isError: true };` | 返回结构化 HarnessError |

### 验证方法

1. 单元测试：在 `mcp-server/src/index.test.ts` 中添加测试，验证向服务器发送不存在工具名时返回含 `UNKNOWN_TOOL` 码的 HarnessError
2. `grep -rn "UNKNOWN_TOOL" mcp-server/src/` — 确认除 types.ts 定义外，index.ts handler 中也引用了该常量
3. `npm run test` — 322+ tests pass
