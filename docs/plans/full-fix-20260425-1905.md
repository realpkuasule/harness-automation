## P1 — Contract Compatibility (fix required)

### P1-1: `optimize_error_message` — Input `projectDir` Required Mismatch

| Aspect | Detail |
|--------|--------|
| **Spec** | `projectDir` in `required` list (openapi.yaml:936) |
| **Impl** | `z.string().optional()` (types.ts:497) |
| **Recurrence** | **新发现** — first reported in 1905 |
| **Risk** | Auto-generated clients may skip `projectDir` and get empty `stats`; failing to pass it is technically a spec violation the impl silently accepts |

**Fix: Update spec to match impl** — The tool works correctly without `projectDir` (skips stats recording). Making `projectDir` required in the impl would break the optional-usage pattern.

```yaml
# openapi.yaml — optimize_error_message requestBody
required: []
# Remove projectDir from required; add description note
```

**File**: `docs/api/openapi.yaml:935-937`
**Change**: Remove `projectDir` from the `required` list. Add description note: "projectDir 可选，不传时不记录评价数据"
**Verify**: `yq eval` confirms `required` no longer includes `projectDir`

| Aspect | Detail |
|--------|--------|
| **Spec** | `cost` (1-5), `feedbackSpeed` (1-5), `frequency` (1-5) declared as `type: integer` with description noting range but no `minimum`/`maximum` constraint (openapi.yaml:1079-1087) |
| **Impl** | Types are `number` with no range enforcement, but engine.ts uses `Math.min(5, Math.round(...))` ensuring runtime values are in range |
| **Recurrence** | **新发现** — low-severity housekeeping |
| **Risk** | Auto-generated validators won't enforce 1-5 range |

**Fix: Add `minimum: 1, maximum: 5` constraints to openapi.yaml**.

```yaml
# openapi.yaml — cost, feedbackSpeed, frequency
cost:
  type: integer
  minimum: 1
  maximum: 5
  description: 执行代价（1-5）
feedbackSpeed:
  type: integer
  minimum: 1
  maximum: 5
  description: 反馈速度（1-5，1=最快）
frequency:
  type: integer
  minimum: 1
  maximum: 5
  description: 发生频率（1-5）
```

**File**: `docs/api/openapi.yaml:1079-1087`
**Change**: Add `minimum: 1, maximum: 5` to cost, feedbackSpeed, frequency
