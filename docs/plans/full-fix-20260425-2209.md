## P1 Items

### P1-A: Fix `DepCheckResult.outdated` Type — string[] → {name, current, wanted}[]

**History**: New finding (first appeared in `gap-analysis-20260425-2209.md` §差异 1)
**Source**: `gap-analysis-20260425-2209.md §Gap 1`
**Recurring**: No
**Effort**: ~10 min

**Root cause**: `deps.ts:21` declares `outdated: string[]`. `checkDependencies()` at lines 66-94 parses `npm outdated --json` (which returns `Record<string, {current, wanted, ...}>`) but only extracts `Object.keys()`, discarding the version info. The OpenAPI spec at `openapi.yaml:2305-2315` expects `Array<{name, current, wanted}>` objects.

**Impact**: `init_harness` output field `summary.recommendedActions.outdated` is `string[]` instead of the declared `object[]`. Downstream consumers expecting `outdated[].name/current/wanted` properties will fail.

**Files to modify**:

| # | File | Line(s) | Current | Change |
|---|------|---------|---------|--------|
| 1a | `mcp-server/src/deps.ts` | 21 | `outdated: string[];` | `outdated: Array<{name: string; current: string; wanted: string}>;` |
| 1b | `mcp-server/src/deps.ts` | 66-94 | `outdated: string[]` with `Object.keys(parsed).slice(0, 10)` | Preserve full object from `npm outdated --json` |

**Detailed change for 1b**:

```typescript
// BEFORE (deps.ts:66-94):
let outdated: string[] = [];
// ...
const parsed = JSON.parse(out);
outdated = Object.keys(parsed).slice(0, 10);

// AFTER:
interface OutdatedPackage { name: string; current: string; wanted: string; }
let outdated: OutdatedPackage[] = [];
// ...
const parsed = JSON.parse(out) as Record<string, { current: string; wanted: string }>;
outdated = Object.entries(parsed)
  .slice(0, 10)
  .map(([name, info]) => ({ name, current: info.current, wanted: info.wanted }));
```

**Cascading effects** — verify these are compatible:
- `index.ts:544-550`: `depCheck.outdated` used to build `depInfo` — will now contain objects instead of strings but fits `Record<string, unknown>` → ✅ compatible
- `index.ts:553-558`: `depInfo` is `Record<string, unknown>` — object values are still `unknown` → ✅ compatible

**Verification**:
1. `npm run build` — TypeScript compiles without errors
2. `npx vitest run` — all tests pass (update any test that asserts `outdated` type)
3. Check `deps.test.ts` for outdated-related assertions — may need update if tests mock npm outdated output
