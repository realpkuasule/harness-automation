## Items

### P15-1: Exclude test files from TypeScript compilation

**Why**: `tsconfig.json` uses `include: ["src/**/*"]` which pulls in `*.test.ts` and `__tests__/`/`__fixtures__/` directories. This bloats `dist/` with test artifacts.

**Change**: 
- `mcp-server/tsconfig.json` â€” change `include` to exclude test files

```json
{
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist", "src/**/*.test.ts", "src/__tests__", "src/__fixtures__"]
}
```

**Verification**: `npm run build` â†’ `ls dist/*.test.*` returns empty, `ls dist/__tests__/` empty, `ls dist/__fixtures__/` empty

### P15-2: Add .npmignore for mcp-server

**Why**: Without `.npmignore`, `npm publish` includes everything: tests, source maps, tsconfig, etc.

**Change**: Create `mcp-server/.npmignore`:

```
# Sources
src/
tsconfig.json

# Tests
*.test.*
__tests__/
__fixtures__/

# Maps (production doesn't need them)
*.js.map
*.d.ts.map

# Dev
node_modules/
eslint.config.js
```

**Verification**: `npm pack --dry-run` â€” confirm test files and source maps are excluded

### P15-3: Configure npm publish metadata in package.json

**Why**: Package is missing standard npm metadata fields (`repository`, `bugs`, `homepage`, `files`). No `prepublishOnly` script means publish can happen with unbuilt code.

**Change**: Update `mcp-server/package.json`:
- Add `repository`, `bugs`, `homepage`, `keywords`, `license`
- Add `files` field to explicitly declare what goes into the package
- Add `prepublishOnly` script that runs `npm run build`
- Update `bin` if needed

**Verification**: `npm pack --dry-run` shows correct file list

### P15-4: Add GitHub Actions publish workflow

**Why**: CI only builds+tests. No automated npm publish on tag.

**Change**: Create `.github/workflows/publish.yml` that:
- Triggers on `v*` tag push
- Runs build + test
- Publishes to npm via `npm publish`

**Verification**: Workflow file exists at `.github/workflows/publish.yml`

### P15-5: Add root README.md

**Why**: Project needs a consumer-facing README for users who discover it on npm or GitHub.

**Change**: Create `README.md` at project root with:
- Project overview and purpose
- Quick start (install + skill usage)
- Architecture overview
- Tool reference (link to OpenAPI spec)
- Development guide

**Verification**: README.md exists, renders correctly on GitHub
