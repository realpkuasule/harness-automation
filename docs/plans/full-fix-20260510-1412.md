## @Roles / 权限全量扫描

对 OpenAPI 规范和全部 22 个 MCP 工具进行权限注解扫描：

| 扫描范围 | 结果 |
|----------|------|
| `docs/api/openapi.yaml` 全文 | 无 `@Roles`、`x-roles`、`security`、`securitySchemes` 声明 |
| 所有 22 个 paths | 无 role/permission 注解 |
| `mcp-server/src/**/*.ts` | 无 `@Roles` 装饰器或类似权限机制 |
| MCP 协议层 (`index.ts`) | 无认证/授权逻辑 |

**结论**: 本项目为 **MCP (Model Context Protocol) stdio 传输**的工具接口，非 HTTP REST API。MCP 协议通过客户端工具白名单（allowlist）机制控制权限，而非服务端 @Roles 注解。OpenAPI 规范中的 `servers[0].url: "mcp://localhost"` 和 `x-mcp-note` 已标明此为 MCP 接口。**无需添加 @Roles 注解**。

唯一与权限相关的功能是 `init_harness` 输出中的 `permissions` 字段（建议哪些 MCP 工具加入 allowlist），该字段已在 OpenAPI `InitHarnessSummary` (L2413-2427) 中完整记录。✅ 无缺失。
