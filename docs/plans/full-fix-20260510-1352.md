## P0 — 关键回归立即修复

### P0-1: 修正 `installNote` 在 OpenAPI `InitHarnessSummary` 中的位置 — 从 `permissions` 下移到顶层

**类型**: Schema 回归 — **Phase 8 P8-1 引入**
**根因**: Phase 8 P8-1 修复将 `installNote` 错误地放在了 `InitHarnessSummary.permissions` 子对象内，而非顶层

**文件**: `docs/api/openapi.yaml` L2384-2401

**当前代码**:
```yaml
permissions:
  type: object
  description: 权限配置提示。init_harness 返回的建议工具白名单（P18-6 新增）
  properties:
    note:
      type: string
      description: 提示文本
    command:
      type: string
      description: 配置方法指引
    suggestedTools:
      type: array
      items:
        type: string
      description: 建议加入 allowlist 的 MCP 工具列表
    installNote:
      type: string
      description: 依赖安装耗时提示
```

**修改为**:
```yaml
permissions:
  type: object
  description: 权限配置提示。init_harness 返回的建议工具白名单（P18-6 新增）
  properties:
    note:
      type: string
      description: 提示文本
    command:
      type: string
      description: 配置方法指引
    suggestedTools:
      type: array
      items:
        type: string
      description: 建议加入 allowlist 的 MCP 工具列表
installNote:
  type: string
  description: 依赖安装耗时提示
```

即将 `installNote` 字段从 `permissions` 的 `properties` 内移出，放到 `InitHarnessSummary` 的顶层 `properties`，与 `permissions` 平级。

**验证**: 
- `grep -A3 "installNote" docs/api/openapi.yaml` 应显示 `installNote` 在顶层缩进（与 `files`、`decisions`、`permissions` 同级），而非在 `permissions` 子属性内
- 对照 `index.ts:706` — `summary.installNote` 是顶层字段

## P1 — 输出 Schema 补全

### P1-1: `rollback` list 模式输出文档化

**类型**: OpenAPI 缺失 — **新发现**
**根因**: `rollback` 的 `list=true` 模式返回不同形状的输出，但 OpenAPI 仅定义了 restore 模式的 `RollbackOutput`

**文件**: `docs/api/openapi.yaml` L328-350

**当前状态**: Response 仅引用 `RollbackOutput`:
```yaml
responses:
  "200":
    description: 回滚结果
    content:
      application/json:
        schema:
          $ref: "#/components/schemas/RollbackOutput"
```

**改动**: 
1. 在 `components/schemas` 中新增 `RollbackListOutput`:
```yaml
RollbackListOutput:
  type: object
  properties:
    backups:
      type: array
      items:
        type: object
        properties:
          id:
            type: string
          files:
            type: array
            items:
              type: string
          createdAt:
            type: string
```

2. 将 `/tools/rollback` 的 200 response 改为条件说明（或在 description 中标注 list=true 时的响应格式）:
```yaml
responses:
  "200":
    description: >
      回滚结果。当 list=true 时返回 RollbackListOutput（备份列表），
      否则返回 RollbackOutput（恢复结果）。
    content:
      application/json:
        schema:
          oneOf:
            - $ref: "#/components/schemas/RollbackOutput"
            - $ref: "#/components/schemas/RollbackListOutput"
```

**验证**:
- `grep "RollbackListOutput" docs/api/openapi.yaml` 返回结果
- 对照 `index.ts:817-824` — list 模式返回 `{ backups: [...] }`

### P1-2: `init_harness` dryRun 输出变体 Schema 标注

**类型**: Schema 精度 — **新发现**
**根因**: dryRun=true 时返回精简的 summary（不含 `backupDir`/`recommendedActions`/`huskySetup`/`permissions`/`installNote`），且不含 `validation`/`errors`，但 OpenAPI Schema 将所有字段标为始终存在

**文件**: `docs/api/openapi.yaml` L259-273

**当前状态**: Response schema 无条件列出所有字段:
```yaml
schema:
  type: object
  properties:
    files: ...
    summary:
      $ref: "#/components/schemas/InitHarnessSummary"
    validation:
      $ref: "#/components/schemas/ValidateSetupOutput"
    errors: ...
```

**改动**: 在 `/tools/init_harness` 的 response description 中明确标注 dryRun 差异（已在 L235-236 有描述，但需补充具体字段差异），并考虑将 `validation` 和 `errors` 标注为 "仅在非 dryRun 时返回"。

具体修改 L253-273 response description:
```yaml
responses:
  "200":
    description: >
      初始化结果。非 dryRun 模式返回完整的 files/summary/validation/errors；
      dryRun=true 时仅返回 files（空数组）和 summary（仅含 files 和 decisions 字段，
      不含 backupDir/recommendedActions/huskySetup/permissions/installNote），
      且不含 validation/errors 字段。
```

同时更新 L235-236 的 endpoint description 中的 dryRun 说明，明确列出被省略的字段。

**验证**:
- `grep "dryRun" docs/api/openapi.yaml | grep -i "不含\|without\|omitted"` 返回更新后的说明
- 对照 `index.ts:492-506` — dryRun 仅返回 `{ files: [], summary: { files, decisions } }`
