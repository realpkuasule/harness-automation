## P0 — 核心行为偏差

### GAP-2: Medium 枚举粒度不足（首次）

**缺口**: `types.ts:8` 定义 5 值，设计 §3.1.3 要求 8 值。缺 `linter_error`/`linter_warn`/`linter+hook`/`none`。

**影响**: ESLint 生成器无法从 Medium 推断规则级别；无法表达复合介质策略；无法禁用规则。

**当前绕过**: `generators/eslint.ts:25-34` 已有 per-rule 硬编码 error/warn 映射。

**改动**:

1. **`src/types.ts:8`** — Medium 扩展为 `"claude.md" | "settings.json" | "linter" | "linter_error" | "linter_warn" | "linter+hook" | "hook" | "ci" | "none"`
   - 保留 `"linter"` 作为向后兼容别名，新代码应使用具体级别
   - `Medium` 类型改为联合类型而非枚举，避免破坏现有 switch exhaustiveness

2. **`src/engine.ts:305-350`** — `_finalDecision()` 扩展决策逻辑：
   - 行 305 前插入 `_specialCases()` 调用（见 GAP-14），匹配到特殊规则时直接返回
   - 当前 `_finalDecision()` 返回值映射：`linter → linter_error`（高 cost/高 freq）、`linter_warn`（低 cost）
   - 增加 `"linter+hook"` 返回路径：当 `category="security" && cost >= 3 && feedbackSpeed <= 2`
   - 增加 `"none"` 返回路径：当 `cost >= 4 && frequency <= 2`

3. **`src/generators/eslint.ts:12-49`** — 移除 per-rule 硬编码 `ruleMap`，改为从 `recommendedMedium` 推导级别：
   ```typescript
   const severity = d.recommendedMedium === "linter_error" ? "error"
     : d.recommendedMedium === "linter_warn" ? "warn"
     : d.recommendedMedium === "linter" ? "error" // 向后兼容
     : undefined;
   if (!severity) continue; // 非 linter 规则跳过
   ```

4. **`src/index.ts:1087-1111`** — `generateProjectFiles()` 过滤逻辑：`recommendedMedium` 包含 `"linter"` 前缀的均触发 eslint 生成

**验证方法**: evaluate_rules 对 security 高代价规则返回 `linter_error` 或 `linter+hook`；对低代价规则返回 `linter_warn`。eslint.config.json 中规则级别正确反映（error vs warn）。

```typescript
export function shouldAutoTrigger(
  recentErrors: string[]
): CognitiveRequest | null {
  // 检测重复错误模式：同一规则在最近 3 次调用中出现 2 次以上
  const errorCounts = new Map<string, number>();
  for (const err of recentErrors) {
    errorCounts.set(err, (errorCounts.get(err) ?? 0) + 1);
  }
  for (const [ruleId, count] of errorCounts) {
    if (count >= 2) {
      return {
        skillType: "educational",
        ruleId,
      };
    }
  }
  return null;
}
```

**验证方法**: `shouldAutoTrigger` 在重复错误时返回 educational 请求。
