### P0-1: init_harness 输入必填性不一致（新发现）

| 维度 | 值 |
|------|-----|
| **缺口** | OpenAPI 声明 `InitHarnessInput` 仅 `projectDir` 必填，但 Zod Schema 要求 `projectPhase`/`teamSize`/`techStack` 全部必填 |
| **来源** | 1430 H1 |
| **历史** | **新发现** |

**当前状态**:

- OpenAPI (`docs/api/openapi.yaml:1142`): `required: [projectDir]` — 仅 1 个必填字段
- Zod (`mcp-server/src/types.ts:336-347`): `projectPhase`/`teamSize`/`techStack` 均为无 `.optional()` 的断言

**影响**: 按 OpenAPI 生成的客户端在调用 `init_harness` 时如果只传 `projectDir`，会收到 Zod 校验错误。

**改动方案**（两种选择）：

**方案 A** — 修改 OpenAPI 匹配 Zod（推荐）:
- `docs/api/openapi.yaml:1142` 将 `required: [projectDir]` 改为：
  ```yaml
  required: [projectDir, projectPhase, teamSize, techStack]
  ```
- 同步更新 `projectPhase`/`teamSize`/`techStack` 的 description 去掉"可选"暗示
- 1 个文件改动

**方案 B** — 修改 Zod 匹配 OpenAPI:
- `mcp-server/src/types.ts:338-340` 三个字段改为 `.optional()`:
  ```typescript
  projectPhase: z.enum([...]).optional().describe(...),
  teamSize: z.enum([...]).optional().describe(...),
  techStack: z.array(z.enum([...])).optional().describe(...),
  ```
- `mcp-server/src/index.ts:549-553` (handler) 添加 fallback 逻辑：
  ```typescript
  if (input.preset) {
    input.projectPhase = input.projectPhase ?? input.preset.projectPhase ?? "early";
    input.teamSize = input.teamSize ?? input.preset.teamSize ?? "small";
    input.techStack = input.techStack ?? input.preset.techStack ?? ["generic"];
  }
  ```
  (该 fallback 已存在，index.ts:549-553 行已有相同逻辑，所以仅改 Zod 即可)
- 2 个文件改动

**推荐: 方案 A** — 改动更少，且实际使用场景中确实总是需要这三个参数。

**验证方法**: 
- 调用 `init_harness` 不传 `projectPhase` → Zod 报错 (方案 A 不变)
- 或者不传 `projectPhase` 但传 `preset.projectPhase` → fallback 生效 (方案 B)
- OpenAPI schema 校验通过
